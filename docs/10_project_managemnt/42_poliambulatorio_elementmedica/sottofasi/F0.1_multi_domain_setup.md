## 📋 MICRO-TASK

### F0.1.1 - Configurazione DNS e Domini (4 ore)

**Obiettivo**: Configurare DNS per separare i due frontend

**Attività**:
1. Verificare proprietà domini
2. Configurare record DNS:
   - `elementmedica.com` → IP server
   - `www.elementmedica.com` → CNAME a elementmedica.com
   - `app.elementmedica.com` → IP server (frontend privato)
   - `elementformazione.com` → IP server (già esistente)
3. Configurare TTL appropriati (300s per testing, 3600s per produzione)
4. Verificare propagazione DNS

**File da Modificare**:
- Nessun file (configurazione DNS provider esterno)

**Test di Accettazione**:
- [ ] `dig elementmedica.com` restituisce IP corretto
- [ ] `dig app.elementmedica.com` restituisce IP corretto
- [ ] Propagazione DNS completata (< 1 ora)

**Comandi Verifica**:
```bash
# Verifica DNS
dig +short elementmedica.com
dig +short app.elementmedica.com
dig +short www.elementmedica.com

# Verifica propagazione globale
curl -s "https://dns.google/resolve?name=elementmedica.com&type=A" | jq
```

### F0.1.2 - Certificati SSL Multi-Domain (3 ore)

**Obiettivo**: Generare certificati SSL per tutti i domini

**Attività**:
1. Installare/aggiornare Certbot
2. Generare certificato wildcard o multi-domain:
   ```bash
   sudo certbot certonly --nginx \
     -d elementmedica.com \
     -d www.elementmedica.com \
     -d app.elementmedica.com
   ```
3. Configurare auto-renewal
4. Verificare certificati

**File da Modificare**:
- `nginx/sites-available/elementmedica.conf` (nuovo)
- `nginx/sites-available/elementmedica-app.conf` (nuovo)

**Nginx Config Template**:
```nginx
# nginx/sites-available/elementmedica.conf
server {
    listen 80;
    server_name elementmedica.com www.elementmedica.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name elementmedica.com www.elementmedica.com;
    
    ssl_certificate /etc/letsencrypt/live/elementmedica.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/elementmedica.com/privkey.pem;
    
    # Frontend pubblico
    location / {
        proxy_pass http://localhost:5173;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_cache_bypass $http_upgrade;
    }
    
    # API
    location /api {
        proxy_pass http://localhost:4003;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

```nginx
# nginx/sites-available/elementmedica-app.conf  
server {
    listen 80;
    server_name app.elementmedica.com;
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name app.elementmedica.com;
    
    ssl_certificate /etc/letsencrypt/live/elementmedica.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/elementmedica.com/privkey.pem;
    
    # Frontend privato (medica)
    root /var/www/elementmedica-app/dist;
    index index.html;
    
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    # API
    location /api {
        proxy_pass http://localhost:4003;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

**Test di Accettazione**:
- [ ] `https://elementmedica.com` carica senza warning SSL
- [ ] `https://app.elementmedica.com` carica senza warning SSL
- [ ] Certificato valido per almeno 60 giorni
- [ ] Auto-renewal configurato

**Comandi Verifica**:
```bash
# Verifica certificato
openssl s_client -connect elementmedica.com:443 -servername elementmedica.com < /dev/null 2>/dev/null | openssl x509 -noout -dates

# Verifica auto-renewal
sudo certbot renew --dry-run
```

### F0.1.3 - Test Routing Domini (2 ore)

**Obiettivo**: Verificare che il routing funzioni correttamente per entrambi i domini

**Attività**:
1. Creare pagina test per ogni frontend
2. Verificare routing corretto
3. Verificare isolamento (dominio A non accede a contenuti B)
4. Documentare configurazione

**Test di Accettazione**:
- [ ] `https://elementmedica.com` → Frontend pubblico (landing)
- [ ] `https://app.elementmedica.com` → Frontend privato (login medica)
- [ ] `https://elementformazione.com` → Frontend formazione (esistente)
- [ ] API accessibili da entrambi i domini
- [ ] CORS configurato correttamente

**Test Manuali**:
```bash
# Test frontend pubblico
curl -I https://elementmedica.com

# Test frontend privato
curl -I https://app.elementmedica.com

# Test API da dominio medica
curl -I https://app.elementmedica.com/api/v1/health

# Test CORS headers
curl -I -X OPTIONS https://app.elementmedica.com/api/v1/health \
  -H "Origin: https://app.elementmedica.com" \
  -H "Access-Control-Request-Method: GET"
```
